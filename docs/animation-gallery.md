# Animation Gallery — Design Document

**Status:** Design / Wishlist  
**Created:** 2026-03-29

---

## Problem

The current gallery (`docs/gallery/`) contains static SVG snapshots of
render effects. Animation effects (A01–A11+) cannot be represented as a
single frame — they require time.

This document describes how to build a gallery for animated effects.

---

## Format Comparison

| Format | Quality | File size | GitHub inline | Browser | Terminal | Deps |
|--------|---------|-----------|---------------|---------|----------|------|
| Animated SVG (SMIL) | Good | Large | ✅ | ✅ | ❌ | None |
| APNG | Excellent | Medium | ✅ | ✅ | ❌ | Pillow |
| GIF | OK (256 color) | Small | ✅ | ✅ | ❌ | Pillow |
| HTML+CSS | Excellent | Medium | ❌ | ✅ | ❌ | None |
| asciinema .cast | Perfect (terminal) | Tiny | ❌ | ✅* | ✅ | None |
| MP4/WebM | Best | Tiny | ❌ | ✅ | ❌ | ffmpeg (system) |

*via asciinema player web component

## Recommended Architecture: Two-Tier Gallery

### Tier 1 — Static gallery (exists, keep as-is)

`docs/gallery/` — SVG snapshots of non-animated effects  
Generated by `scripts/generate_gallery.py`  
No changes needed.

### Tier 2 — Animation gallery (new)

`docs/anim_gallery/` — APNG files of animated effects  
Generated by `scripts/generate_anim_gallery.py`  
Linked from `docs/anim_gallery/README.md` (same structure as static gallery)

## Primary Format: APNG

**Why APNG over GIF:**
- 24-bit color (vs GIF's 256-color palette) — ANSI true-color survives
- Alpha channel — transparent background possible
- Universally supported in modern browsers
- Pillow supports APNG natively (no extra deps beyond existing Pillow use)
- Auto-plays in GitHub README and browser

**Why APNG over animated SVG:**
- A 40×10 grid × 30 frames = 12,000 SVG elements → slow to render/load
- APNG file size is smaller for complex animations
- SVG-SMIL support is inconsistent across browsers

**APNG generation pipeline:**
```
frame_generator()           # yields str (ANSI) per frame
  → ansi_parser.parse()     # strip ANSI → (char, color) per cell
  → frame_to_pil_image()    # render cell grid to PIL Image (new module)
  → PIL.Image.save(apng)    # stitch frames into APNG
```

---

## Secondary Format: asciinema .cast

For the "authentic terminal" experience — APNG looks like a screenshot of a
terminal, `.cast` files play back *as* a terminal.

**Format:** JSON lines, trivially generated from frame strings:
```python
# asciinema v2 format
header = {"version": 2, "width": cols, "height": rows, "title": "..."}
events = [(timestamp, "o", frame_ansi_string), ...]
```

No external deps — just write JSON. Playback via:
- `asciinema play file.cast` (CLI)
- `<asciinema-player>` web component (docs site)
- `https://asciinema.org/` (upload for sharing)

**Target:** generate `.cast` alongside APNG for every animation effect.

## New Module: `justdoit/output/apng.py`

```python
def frames_to_apng(
    frames: list[str],          # list of ANSI frame strings
    path: str,                   # output file path
    fps: float = 12.0,           # playback speed
    font_size: int = 14,         # px, same as SVG output
    bg_color: str = "#111111",   # background
    loop: int = 0,               # 0 = infinite loop
) -> None:
    """Render a list of ANSI frame strings to an animated PNG file.

    Requires Pillow. Raises ImportError with install hint if not available.
    Each frame is rendered as a PIL Image using monospace character grid.
    Frames are stitched into an APNG with the given fps.
    """
```

**Requires Pillow** — same optional-dep pattern already established.
Gate with `pytest.importorskip("PIL")` in tests.

---

## New Module: `justdoit/output/cast.py`

```python
def frames_to_cast(
    frames: list[str],          # list of ANSI frame strings
    path: str,                   # output .cast file path
    fps: float = 12.0,
    title: str = "JustDoIt",
    cols: int = 80,
    rows: int = 24,
) -> None:
    """Write an asciinema v2 .cast file from a list of ANSI frame strings.

    Pure Python stdlib — no external dependencies.
    """
```

**No deps** — pure Python JSON. Ship in core, not gated.

## New Script: `scripts/generate_anim_gallery.py`

Parallel to `generate_gallery.py`. Runs all registered animation effects,
collects frames, saves APNG + `.cast` per effect.

```
docs/anim_gallery/

  A11-transporter-tng.apng
```

## Implementation Order

1. `justdoit/output/cast.py` — pure Python, no deps, fastest to ship
2. `justdoit/output/apng.py` — Pillow dep, core format
3. `scripts/generate_anim_gallery.py` — wire it all together
4. `docs/anim_gallery/README.md` — auto-generated index
5. `.claude/skills/regenerate-anim-gallery/SKILL.md`
6. Update `CLAUDE.md` to reference anim gallery skill

Ship in this order so `.cast` files exist even before APNG is ready.

## Future: Live Gallery Site

Long term, `docs/` could be a GitHub Pages site with:
- Static SVG gallery (exists)
- APNG animation gallery (this doc)
- Embedded asciinema players for `.cast` files
- A live terminal emulator (xterm.js) for interactive demo

The `asciinema-player` web component drops in with one `<script>` tag.
No backend required — all static files.
